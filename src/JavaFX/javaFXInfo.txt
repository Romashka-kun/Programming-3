#Pane

Панели позволяют распологать на себе другие элементы интерфейса. Например, кнопки, поля ввода и другие панельки.
Другие виды панелей (VBox, HBox, GridPane и т.п.)

#Constraints

Ограничения. Они позволяют указать параметры расположения элементов внутри панели. Т.е. панель при расположении
элементов руководствуется дополнительными пожеланиями о расположении, взятыми из constraints.
Пример constraints - Insets, соответствует margin, padding в CSS. Другой пример, в HBox элементы располагаются по
горизонтали друг за другом, а с помощью constraint можно указать выравнивание по вертикали.
У каждого вида панели свой набор всевозможных constraint.

#Preferred size

Предпочитаемый размер компонентов.
Каждый Region знает размер, который ему хочется иметь. Это размер, который позволяет отобразить всё своё содержимое.
Например, для кнопки предпочитаемый размер соответствует тексту и шрифту кнопки.
У панелей предпочитаемый размер вычисляется на основе предпочитаемых размеров содержимого. В этом размере помещается
всё содержимое панели, и не остаетс лишнего пространства.
Компонент необязательно имеет размер, соответствующий Preferred size.
Ещё бывают размеры MinimalSize и MaximalSize, компоненты никогда не будут меньше или больше этих размеров.

#BorderPane

Позволяет расположить пять узлов: сверху, снизу, справа, слева, по центру.

Constraints:
alignment - где расположен элемент, если он не растянут на всю область. Снизу слева, снизу справ, по центру и т.п.
margin - отступы
какой из пяти - это constraint
указывается кодом:
    BorderPane.setLeft(...);
    BorderPane.setCenter(...);
    ...

#FlowPane

Элементы располагаются друг за другом слева направо, а если место по горизонтали кончается, элемент переносится на
следующую "строку".
Можно устанавливать промежутки между элементами hgap, vgap.
Можно выкладывать элементы не по горизонтали, а по вертикали: orientation.

#TilePane

#StackPane
